# WEB REDESIGN PLAYBOOK

## 适用场景
适用于所有网页改版、UI优化、视觉升级、内容型页面重构任务。

## 总体目标
把页面做得更成熟、更克制、更国际化、更有设计感，而不是更可爱。

## 风格关键词
- warm
- quiet
- intimate
- editorial
- restrained
- layered
- cinematic

## 明确反对的方向
- cute
- childish
- pet-homepage
- sticker-like
- overly romantic
- diary-like

## 页面常见问题排查
每次诊断时，必须优先检查：
- 页面是否低幼
- 首屏是否喧宾夺主或过空
- 桌面端是否版心失衡
- 是否存在内容全挤在左侧、右侧大面积空白
- 是否只有留白，没有构图
- 是否缺少视觉锚点
- 是否所有模块权重都一样弱
- 是否过冷、过平、过样板
- 是否缺少温度和层次
- 是否没有动效导致页面死板

## 布局原则
- 桌面端主容器默认 max-width 建议 1180px~1240px
- 页面内容必须水平居中
- 左右 padding 需要合理
- Hero 优先双栏
- 不允许所有 section 只堆在左列
- Featured / Timeline / Notes 需要有明显权重区别
- 留白必须服务于构图，而不是空

## Hero 原则
Hero 必须同时承担：
- 建立页面气质
- 提供明确视觉重心
- 告诉用户这是一个什么页面
- 提供至少一个明确 CTA

避免：
- 情绪过满
- 文案喊话过多
- 只有标题和按钮，没有画面
- 过空导致首屏无吸引力

## 温度原则
温度来自于：
- 更暖的中性色层次
- 柔和纸张感
- 轻微材质感
- 暖色高光
- 克制但有人味的文案
- 视觉锚点

温度不来自于：
- 粉色
- emoji
- 卡通贴纸
- 低幼措辞
- 宠物主页语气

## 文案原则
文案应当：
- 成熟
- 克制
- 有余味
- 有旁白感
- 有一点人味

避免：
- "全世界"
- "小猫咪"
- 过度甜腻表达
- 过度功能化、冷冰冰的介绍

## 模块原则
- Featured 应该成为主内容焦点之一
- All Tracks 不能像后台列表
- Timeline 不能像备注列表
- Notes 不能只是碎卡片堆叠
- 各模块必须在结构、大小、节奏上有所区别

## 设计判断标准
优秀结果应当呈现为：
- 高级但不冷
- 温暖但不幼
- 克制但不空
- 有设计感但不装腔
- 有动效但不炫技
